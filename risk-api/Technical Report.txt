3.2 CHALLENGE: CASCADE DELETE FOREIGN KEY VIOLATIONS
------------------------------------------------------

PROBLEM:
DELETE /companies/ endpoint failed with:
"ForeignKeyViolation: foreign key constraint financial_metrics_company_id_fkey"

ROOT CAUSE:
When deleting company, related records in financial_metrics and stock_prices 
still referenced the company_id foreign key.

SOLUTION 1: SQLAlchemy Relationship Cascade
Configure cascade in models.py relationships:

class Company(Base):
    financial_metrics = relationship(
        "FinancialMetric",
        back_populates="company",
        cascade="all, delete-orphan"  # KEY FIX
    )

SOLUTION 2: ORM Delete Loop
Changed main.py delete endpoint from raw SQL to ORM:

BEFORE (Broken):
db.query(models.Company).delete()  # Raw SQL, bypasses cascade

AFTER (Fixed):
companies = db.query(models.Company).all()
for company in companies:
    db.delete(company)  # ORM delete, respects cascade

RESULT:
DELETE operations now automatically remove related records in single 
transaction. Tested with 150 companies + 10,000+ price records - success.

LESSON LEARNED:
1. CASCADE DELETE configuration is critical for hierarchical data
2. ORM operations respect relationship configurations, raw SQL doesn't
3. Always test delete operations with real foreign key relationships

3.3 CHALLENGE: NUMPY TYPE SERIALIZATION
-----------------------------------------

PROBLEM:
API crashed with:
"TypeError: Object of type float64 is not JSON serializable"

ROOT CAUSE:
Yahoo Finance returns numpy.float64/int64 types which FastAPI cannot 
serialize to JSON.

SOLUTION:
Conversion helper function applied at data ingestion:

def convert_numpy(value):
    if value is None or pd.isna(value):
        return None
    if isinstance(value, (np.integer, np.floating)):
        return float(value)
    return value

USAGE:
financial = FinancialMetric(
    pe_ratio=convert_numpy(info.get('trailingPE')),
    eps=convert_numpy(info.get('trailingEps'))
)

RESULT:
Zero serialization errors across all 150 companies and 22 endpoints.

LESSON LEARNED:
Convert external API data types to Python natives at ingestion, not response.

3.4 CHALLENGE: DATABASE SESSION MANAGEMENT
-------------------------------------------

PROBLEM:
Early implementation had connection leaks causing "too many connections" 
errors after 20-30 imports.

SOLUTION:
Session-per-company pattern with proper cleanup:

for symbol in symbols:
    db = SessionLocal()  # Fresh session
    try:
        import_company_data(symbol, db)
        import_financial_data(symbol, db)
        # Each function commits internally
    except Exception as e:
        db.rollback()  # Isolated rollback
    finally:
        db.close()  # Always clean up

RESULT:
Imported 150 companies with zero connection errors. Errors in one company 
don't affect subsequent imports.

LESSON LEARNED:
Use fine-grained session scopes (one per item) for better error isolation 
and resource management.
